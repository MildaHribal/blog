Small products still need fast incident diagnosis.

## Minimal stack that works

- structured logs
- one error tracker
- uptime check
- two dashboards: latency + errors

## Logging rule

Every request should have a request id in logs and downstream calls.

## Start with these alerts

- error rate > baseline
- p95 latency spike
- restart loop detected

You can run this cheaply and still debug most issues within minutes.
