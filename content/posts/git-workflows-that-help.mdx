I used Git for two years before I learned anything beyond `add`, `commit`, `push`, and `merge`. Then I started working on a bigger codebase and discovered that Git is full of tools I'd been avoiding.

## Stash with a name

```bash
git stash push -m "half-done login refactor"
```

Plain `git stash` is fine until you have three stashes and forget what's in any of them. Named stashes are trivially better.

```bash
git stash list
# stash@{0}: On main: half-done login refactor
git stash pop stash@{0}
```

## Interactive rebase before pushing

```bash
git rebase -i HEAD~4
```

This opens an editor with the last 4 commits. You can:
- `pick` — keep as is
- `squash` — merge with the previous commit
- `reword` — change the message
- `drop` — remove entirely

Cleaning up "WIP", "fix", and "oops" commits before pushing makes the history actually useful.

## Bisect to find a regression

```bash
git bisect start
git bisect bad          # current commit is broken
git bisect good v1.4.0  # this tag was working
```

Git checks out a commit halfway between. You test, mark it `good` or `bad`, and repeat. Binary search through history to find exactly when a bug was introduced. Surprisingly fast even over hundreds of commits.

## The fixup workflow

Made a mistake two commits ago?

```bash
# Make your fix, then:
git commit --fixup HEAD~1
git rebase -i --autosquash HEAD~3
```

The fixup commit automatically gets squashed into the right place. Clean history without manual editing.

## Reflog: your safety net

```bash
git reflog
```

This shows every state HEAD has been in, including commits you thought you lost. If you did a bad rebase, reset to the wrong thing, or deleted a branch prematurely — it's in the reflog.

```bash
git checkout HEAD@{3}
```

Go back to where you were three operations ago.

## Amend when you haven't pushed

```bash
git commit --amend --no-edit
```

Forgot to stage a file? Spotted a typo in the commit message? `--amend` rewrites the last commit. Only use it before you push — after pushing, it rewrites history and causes trouble for anyone who pulled.

These are all standard Git. They're just not in the intro tutorials because those tutorials end at "push to GitHub."
