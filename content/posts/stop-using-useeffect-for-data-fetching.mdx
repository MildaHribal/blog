Every React developer has written this at some point:

```ts
useEffect(() => {
  fetch("/api/posts")
    .then((r) => r.json())
    .then((data) => setPosts(data));
}, []);
```

It works. It also silently breaks in at least three different ways.

## The race condition you're not seeing

When the user navigates fast — or your component remounts for any reason — both fetches are in flight. The slower one wins. You end up rendering stale data with no error, no warning, and no way to reproduce it in dev mode.

The fix if you're stuck with `useEffect`:

```ts
useEffect(() => {
  let cancelled = false;

  fetch("/api/posts")
    .then((r) => r.json())
    .then((data) => {
      if (!cancelled) setPosts(data);
    });

  return () => {
    cancelled = true;
  };
}, []);
```

Nobody writes this. I didn't write this for two years.

## The loading state lie

Your component shows a spinner, then shows data. But what about the 200ms between mount and the first render where `posts` is `undefined`? If a child reads `posts.length`, that's a crash waiting for a slow connection.

Defensive code helps:

```ts
const [posts, setPosts] = useState<Post[]>([]);
const [loading, setLoading] = useState(true);
```

But now you have three pieces of state that need to stay in sync. Add an `error` state and you're managing a mini state machine by hand.

## The real fix: move it to the server

If you're on Next.js App Router, the simplest solution is to not fetch client-side at all:

```tsx
// app/posts/page.tsx
export default async function PostsPage() {
  const posts = await getPosts(); // runs on the server

  return <PostList posts={posts} />;
}
```

No race condition. No loading state. No `useEffect`. No stale data. The component renders with data already there.

If you need client-side fetching — polling, user-triggered refreshes, mutations — use a library like SWR or React Query. They handle cancellation, deduplication, revalidation, and error states properly.

`useEffect` + `fetch` is a two-liner that hides a 50-liner of edge cases. There are better tools.
