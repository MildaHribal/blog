Memory leaks in Node.js are usually boring, not exotic. Most were introduced by "temporary" helpers that never got cleaned up.

## 1) Global maps with no eviction

A `Map` used as request cache stayed alive for weeks. Traffic rose, heap grew, pod restarted.

```ts
const cache = new Map<string, UserProfile>();
```

Fix: add TTL + max size, or use Redis.

## 2) Event listeners never removed

Long-lived objects attached listeners on each request.

```ts
emitter.on("done", handler);
```

Fix: unsubscribe after completion, or use `.once` where possible.

## 3) Unbounded logging buffers

A custom logger buffered too much when transport was slow.

Fix: drop old entries, add backpressure, and track queue length metrics.

## Practical checklist

- Track heap usage over time, not single snapshots
- Alert on restart frequency + OOM kills
- Add p95 response time and GC pause charts side by side

Most leaks are architecture leaks, not syntax mistakes.
