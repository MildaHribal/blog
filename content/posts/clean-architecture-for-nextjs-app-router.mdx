App Router gives flexibility, but without boundaries things get messy fast.

## Layers I keep

- `app/` for route composition
- `components/` for reusable UI
- `lib/` for domain and infra helpers
- `content/` for static source data

## Rule that helps most

Route files orchestrate. Business logic lives in `lib/`.

## Benefits

- easier testability
- easier refactoring
- smaller server component files

Good structure is not about purity. It is about reducing future decision fatigue.
