Most dashboards I build do not need full duplex transport.

## Use SSE when

- server pushes updates only
- reconnection should be simple
- infra should stay HTTP-friendly

## Use WebSockets when

- client sends frequent realtime messages
- low-latency bidirectional flow is required
- protocol overhead is acceptable

## Operational reality

SSE is easier to debug with existing tooling and proxies. WebSockets can be great, but they increase complexity quickly.

Start with SSE unless product requirements prove otherwise.
