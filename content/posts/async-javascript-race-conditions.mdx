Race conditions appear when stale async results overwrite fresh state.

## Classic example

User types quickly, request A starts, request B starts, B returns first, then A returns and clobbers UI with old data.

## Safer patterns

- abort old requests
- track request tokens
- ignore responses from stale tokens

```ts
const controller = new AbortController();
fetch(url, { signal: controller.signal });
controller.abort();
```

## In React

Use cleanup in effects and centralize async state transitions.

Correctness first, then micro-optimizations.
