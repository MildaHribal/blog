Form code gets messy when validation logic is duplicated in three places.

## My structure

- UI validation for immediate feedback
- Shared schema for API payload checks
- Server-side validation as source of truth

## Keep the UI simple

Validate on blur for most fields, on input only for tiny constraints like max length.

## Shared schema wins

A single schema for request parsing removes drift between frontend and backend expectations.

## Error handling UX

- show one clear message per field
- keep submit button enabled unless request is active
- preserve typed values on failure

Predictable form behavior beats clever form abstractions every time.
