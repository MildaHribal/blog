Versioning is a communication problem first.

## What I do

- keep a stable contract document
- add fields before removing fields
- mark deprecations with a clear sunset date
- monitor usage per version

## Header vs path

Path versioning (`/v1`) is explicit and easy for clients. Header versioning looks cleaner but is harder to debug in proxies and logs.

## Migration strategy

- release `v2`
- dual-run `v1` and `v2`
- notify clients with concrete examples
- remove `v1` only after observed zero critical traffic

Breaking clients is usually avoidable with enough observability.
