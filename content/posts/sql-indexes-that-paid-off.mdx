I stopped adding indexes "just in case". I now add them only after checking real query patterns.

## Case 1: status + createdAt feed

```sql
CREATE INDEX idx_posts_status_created_at ON "Post" (status, "createdAt" DESC);
```

That matched the exact filter and ordering path used by the feed.

## Case 2: unique lookup by slug

```sql
CREATE UNIQUE INDEX idx_posts_slug ON "Post" (slug);
```

No full scans, deterministic lookup.

## Mistakes I stopped making

- indexing every column from ORM model
- ignoring sort order in index definition
- skipping `EXPLAIN ANALYZE`

## Rule of thumb

Index the query, not the table.
